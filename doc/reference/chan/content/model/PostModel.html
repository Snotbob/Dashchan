Class Overview

Extendable model containing post data.

For the first you must override the following methods:

MethodDescription
getParent()Return number of parent thread. If this is original post you must return null, "0" or the same value as post number.
getNumber()Return unique post number.
getTimestamp()Return a time when post was written.
getComment()Return a comment of post.
setComment(String)Sometimes client can update comment field. You must provide your comment field updating on this method.

You also can override the rest methods. Read their description to understand the rules.

You must mark all editable fields as Editable. Usually you must mark all field, excepting these ones:

FieldException
Post numberThe number is unique value. If this value changes, it begins to be seen as another post.
Parent numberThe number of thread is unique thread. New number means new thread.
CommentThis field will be checked anyway. If you mark this field, this field will be checked twice.
FilesFiles will be checked anyway without editable mark.
