Class Overview

Provides extension configuration.

During construction you can request the following features using requestFeature(int) and requestFeatureWithCondition(int, String, boolean) methods:

  • FEATURE_HTTPS
  • FEATURE_PARTIAL_THREAD_LOADING
  • FEATURE_SEND_POST
  • FEATURE_SEARCH
  • FEATURE_CATALOG
  • FEATURE_SINGLE_BOARD_MODE
  • FEATURE_READ_SINGLE_POST
  • FEATURE_CAPTCHA_PASS
  • FEATURE_WATCHER
  • FEATURE_UNMARK
  • FEATURE_DELETE_POSTS
  • FEATURE_SEND_REPORT
  • FEATURE_USER_BOARDS
  • FEATURE_POPULAR_THREADS
  • FEATURE_USER_AUTHORIZATION

Static configuration

This configuration remains in RAM only while client is launched. You must configure these settings during construction.

You can configure poster's default name using setDefaultName(String) and setDefaultName(String, String).

You can configure board title and description using setBoardTitle(String, String) and setBoardDescription(String, String).

You can configure bump limit using setBumpLimit(int) and setBumpLimit(String, int). You can change bump limit mode using setBumpLimitMode(BumpLimitMode).

You can configure pages count using setPagesCount(String, int).

You can add supported captchas using addCaptchaType(String).

Dynamic configuration

This configuration can be written to client's preferences and read every time client launches.

You can store poster's default name using storeDefaultName(String, String).

You can store board title and description using storeBoardTitle(String, String) and storeBoardDescription(String, String).

You can store bump limit using storeBumpLimit(String, int).

You can store pages count using storePagesCount(String, int).

You can store any another properties using set(android.content.SharedPreferences.Editor, String, String, boolean), set(android.content.SharedPreferences.Editor, String, String, int) and set(android.content.SharedPreferences.Editor, String, String, String) methods. You can obtain editor with edit().

You can get stored properties using {@link ChanConfiguration#get(String, String, int) and get(String, String, String) methods.

Configuring actions

You can configure captcha using obtainCustomCaptchaConfiguation(String).

You can configure posting using obtainPostingConfiguration(String).

You can configure posts deleting using obtainDeletingConfiguration(String).

You can configure reporting using obtainReportingConfiguration(String).

You can configure captcha pass using obtainCaptchaPassConfiguration().

You can configure authorization using obtainUserAuthorizationConfiguration().

You can configure archivation using obtainArchivationConfiguration().

Additional features

You can store cookies using storeCookie(String, String, String). Later you can get it using getCookie(String).

You can get chan's resources using getResources().

public static final int FEATURE_CAPTCHA_PASS

Allows user to enter authorization data and skip the captcha. With entered data you will receive captchaPass argument. You can perform authorization and save cookies or another data that represents authorization state. Then you can use them to skip captcha.

If you request this feature, you must implement checkCaptchaPass(chan.content.ChanPerformer.CheckAuthorizationData).

public static final int FEATURE_SINGLE_BOARD_MODE

Allows client to use only one board. That exempts you from obligation to override readBoards(chan.content.ChanPerformer.ReadBoardsData) method.

The boardName argument in all methods will be equal to null. You can change this argument using setSingleBoardName(String) method.

public static final int FEATURE_UNMARK

Allows user to get post's markup. With this feature you must be sure that ChanMarkup can do this correctly or provide markup with getCommentMarkup().

public static final int FEATURE_USER_AUTHORIZATION

Allows user to enter authorization data to access some features. You can obtain authorization data using getUserAuthorizationData(). With this data you can perform authorization and save cookies or another data that represents authorization state. Then you can grant user rights to read some threads or something like this.

If you request this feature, you must implement checkUserAuthorization(chan.content.ChanPerformer.CheckAuthorizationData).

public static final int FEATURE_USER_BOARDS

Allows client to download user boards.

If you request this feature, you must implement readUserBoards(chan.content.ChanPerformer.ReadBoardsData).

public static final int FEATURE_WATCHER

Allows client to download posts count.

If you request this feature, you must implement readPostsCount(chan.content.ChanPerformer.ReadPostsCountData).

You should also implement obtainCaptchaPassConfiguration().

public void addCaptchaType (String captchaType)

Add captcha type to list of supported capthas. User may choose captcha in application preferences. Client will obtain configuration of custom captchas with obtainCustomCaptchaConfiguation(String).

There is a list of default captcha types: CAPTCHA_TYPE_RECAPTCHA_1, CAPTCHA_TYPE_RECAPTCHA_2, CAPTCHA_TYPE_YANDEX_TEXTUAL and CAPTCHA_TYPE_YANDEX_NUMERIC. Client may handle these captchas my itself.
