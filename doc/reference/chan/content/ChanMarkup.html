Class Overview

Provides HTML posts handling and post editing.

If your chan supports posting, you must implement obtainCommentEditor(String).

HTML parser uses the following cycle:

1. Calls onStartEnd(HtmlParser, StringBuilder, boolean) with false last argument.
2. Calls handleTagStart(HtmlParser, StringBuilder, String, Attributes) every time parser reaches tag opening.
3. Calls handleTagEnd(HtmlParser, StringBuilder, String) every time parser reaches tag closing.
4. Calls onStartEnd(HtmlParser, StringBuilder, boolean) with true last argument.

At the 1st step you can reset your markup handler state.

The 2nd step provides you information about HTML tag opening. You can check it's name and attributes and then call add(chan.text.HtmlParser.SpanType, Object, int) to add span with specified spanType. This action will save span's start position.

The 3rd step provides you information about HTML tag closing. You chan get last span using getLastSpan() or getLastSpan(chan.text.HtmlParser.SpanType). Then you can close this span with close(int) specifying an end index of span.

If your chan supports FEATURE_UNMARK, you must handle unmark action using isUnmarkMode(). In this case you can deny() spans instead of closing ones. When parser reaches tag opening or closing, you must append necessary markup tags to builder at start and end positions. For example, when strong tag opens, you append **. When strong tag closes, you append ** again.

This class can do the most of these actions by itself. You can configure class behavior using these methods:

  • addTag(String, chan.text.HtmlParser.SpanType)
  • addTag(String, String, chan.text.HtmlParser.SpanType)
  • addTag(String, String, String, chan.text.HtmlParser.SpanType)
  • addBlock(String, boolean)
  • addPreformatted(String)
  • addColorable(String)
